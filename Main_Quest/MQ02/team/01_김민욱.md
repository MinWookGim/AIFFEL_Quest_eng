# 김민욱 — 정상 이미지 주입 · WBF 앙상블 · rtdetr 환경 분석

나는 발표를 맡았고, 개선 실험으로는 세 갈래를 팠다. 병목 두 개(헛경보 30% / 손상 30% 놓침)를 각각 다른 방향으로 공략했다.

## 1. 정상 이미지 주입 — 헛경보 줄이기

baseline은 "무조건 어딘가 손상이 있다"는 전제로만 학습돼서 깨끗한 표면도 손상으로 봤다. 그래서 정상(빈 라벨 = 박스 0개 = 배경) 이미지를 train에 섞어 "문제 없는 경우도 있다"를 가르쳤다. 위조지폐 감별을 배울 때 정상 지폐도 같이 봐야 하는 것과 같다.

yolov8s 하나로 고정하고(변수 격리), 정상 주입량을 0 / 250 / 750 / 1500 / 2500으로 늘리며 곡선을 그렸다. 헛경보는 라벨 없는 정상 200장 기준, recall은 test 301장 기준이다.

| 주입량 | 헛경보율 | damage recall | mAP50 |
|--------|----------|---------------|-------|
| 0 (baseline) | 30% | 0.707 | 0.863 |
| 250 | 25% | 0.670 | 0.862 |
| 750 | 16% | 0.705 | 0.870 |
| 1500 | 17% | 0.722 | 0.869 |
| 2500 | 14% | 0.689 | 0.865 |

헛경보는 30% -> 14%로 계속 줄었고, damage recall은 1500까지는 유지·회복(0.722, 오히려 최고)되다가 2500에서 처음 꺾였다(0.689). 정상을 너무 많이 넣으면 모델이 보수적으로 변해 손상을 놓치기 시작한다는 뜻이다. **적당한 지점(750~1500)이 헛경보를 절반 가까이 줄이면서 recall은 안 내주는 맞교환**이었다.

## 2. WBF 앙상블 — 손상 더 찾기

성격이 다른 모델들의 예측을 WBF(Weighted Boxes Fusion)로 합치면, 서로 놓친 걸 메워 recall이 오를까 봤다. 재학습 없이 추론만 합쳤다.

최종 앙상블 = yolov8s_norm1500(챔피언) + yolo11s / v10s / 12s_norm750 + rtdetr-l.

| 방법 | mAP50 | damage recall |
|------|-------|---------------|
| 단일 최고 | 0.846 | 0.856 |
| 4모델 앙상블 | 0.842 | 0.891 |
| + rtdetr | 0.863 | 0.944 |

손상 recall이 0.856 -> 0.944로 병목을 직격했다. 발산해서 "혼자선 못 쓴다"던 rtdetr을 멤버로 넣으니 오히려 recall·mAP 둘 다 최고였다 — transformer 계열이 YOLO들과 다른 실수를 해서 팀으로는 제 몫을 했다. (다만 mAP50-95는 단일 yolo11s가 근소하게 앞선다. 앙상블의 강점은 "더 많이 찾기"이지 "더 정밀하게 맞추기"는 아니다.)

## 3. rtdetr 환경 한계 분석 — 실패도 자산

rtdetr-l을 로컬 GPU(ROCm)에서 돌리니 학습이 자꾸 nan으로 발산했다. 학습률을 낮춰봤더니 발산 시점이 뒤로 밀리기만 하고(ep19 -> ep24) 결국 터졌다. 즉 **학습률은 발산 시점만 조절할 뿐, 근본 원인은 환경(ROCm 우회의 수치 불안정)** 이었다. 높은 lr은 빨리 배우다 일찍 터지고, 낮은 lr은 느리게 배우다 늦게 터졌다. 어느 쪽이든 발산이었다.

한 에폭만 보고 "극복했다"고 판단할 뻔한 것도 기록으로 남겼다. 실패지만 환경 한계를 실험으로 규명한 셈이다.

## 4. 일반화 검증 (외부 데이터)

우리 앙상블을 낯선 데이터에 붙여봤다. WTBD(Nature 논문 데이터)와 이예령의 blade30 타일화 데이터 둘 다에서 절대 성능은 붕괴했지만(mAP50 0.85 -> 0.02~0.06), 앙상블이 단일보다 손상을 더 찾는 우위는 견고했다. 자세한 건 이예령 파일과 리포트에 있다.
