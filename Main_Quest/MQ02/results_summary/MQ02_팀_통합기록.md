## 발표 3막 서사에 팀 파트 매핑

```
1막 — 문제 + baseline
   EDA(정상 78% 미사용 / Dirt:Damage = 1:15) → 5모델 baseline 비교 → yolov8s 승자("최신≠최고")
   공통 병목 2개: ① damage 30% 놓침(recall~0.70)  ② 깨끗한데 헛경보 30%

2막 — 개선 레버들 (팀이 나눠서 넓게 탐색)
   [헛경보 ↓]  정상 이미지 주입           ← 김민욱
   [recall ↑]  WBF 앙상블(5모델)          ← 김민욱
   [recall ↑]  baseline 앙상블(v8·v12)    ← 박정우      [받을 것]
   [입력품질]  데이터 전처리(타일화)       ← 이예령      (blade30 검증됨)
   [recall↑]   증강 재학습 + 앙상블 백테스트 ← 김민       (노트북3개 읽음)
   [서비스데모] 영상→탐지 웹 서비스          ← 최병철      (Cloud Run 배포·확인됨)
   [환경한계]  rtdetr 발산 분석            ← 김민욱

3막 — 종합 + 최종모델 + 회고
   상호보완: 앙상블(recall↑) ↔ 정상주입(FP↓) = 두 마리 토끼 → 팀 최종모델
   일반화 검증(WTBD): 도메인갭은 크지만 앙상블 우위는 견고
   회고: 실패(rtdetr 환경한계)도 규명하면 자산
```

## A. 문제정의 + baseline (1막) — ✅ 완료

- 데이터: 전체 13,470장 중 라벨 2,995장, **정상(라벨 없음) 10,475장 = 78%**(baseline은 안 씀 → 개선 레버). 박스 **Dirt 581 : Damage 8,770 = 1:15**(Dirt 희귀 → 클래스별 AP 필수). 이미지 586x371 균일.
- split: train 2395 / val 299 / test 301 (Dirt 19% 층화). **전원 동일 설정**(50ep / imgsz640 / batch16 / seed42)로 비교해야 성립.
- 5모델 비교(잣대 A, test 301): **yolov8s 종합 승자**(mAP50 0.863, mAP50-95 0.627, damage recall 0.707 = 안전 관점 1등). "최신(v10/v12/rtdetr)이 항상 낫진 않다."
- 공통 병목: **damage 30% 놓침** + **깨끗한 표면에 헛경보 30%**(confusion에서 배경→damage 과탐 0.98). → 이후 모든 개선의 타깃.
- 자료: `results_summary/1단계_모델비교_종합.md`, `charts/` 그래프, `results_summary/demo/`.

## B. 정상 이미지 주입 — 헛경보 ↓ (2막, 김민욱) — ✅ 완료

- 개념: baseline은 "무조건 어딘가 손상 있다"는 전제로만 학습 → 깨끗한 곳도 손상으로 봄. 정상(빈 라벨=박스 0개=negative) 이미지를 train에 주입해 "문제 없는 경우도 있다"를 인지시킴. (위조지폐 감별=정상지폐도 학습.)
- 비율 곡선 5점(yolov8s 고정=변수 격리, 헛경보=정상 200장, recall=test 301):

| 주입량 | 헛경보율 | damage recall | mAP50 |
|--------|----------|---------------|-------|
| norm0 | 30% | 0.707 | 0.863 |
| norm250 (~9%) | 25% | 0.670 | 0.862 |
| norm750 (~24%) | **16%** | 0.705 | **0.870** |
| norm1500 (~48%) | 17% | **0.722** | 0.869 |
| norm2500 (~51%) | **14%** | 0.689 | 0.865 |

- 발견: 헛경보 30%→14% 계속 감소. recall은 norm1500까지 유지·회복(0.722 최고)하다 **norm2500에서 처음 꺾임(0.689)** = "너무 많은 배경은 모델을 보수화시켜 손상을 놓친다." **sweet spot = norm750~1500.**
- 자료: `normals_fp_curve.png`, `baseline_false_positives.png`(헛경보 데모), `MQ02_종합_히스토리.md` 2단계-A.

## C. WBF 앙상블 5모델 — damage recall ↑ (2막, 김민욱) — ✅ 완료

- 개념: 성격 다른 모델들 예측을 WBF로 합침(재학습 0, 추론만). 서로 놓친 걸 메워 recall↑.
- 결과(잣대 B, test 301): 단일 최고 dmg R 0.856 → **ENS_YOLO(4종) 0.891 → ENS_ALL(+rtdetr) 0.944**, mAP50 0.863(최고).
- 정직한 한계: mAP50-95는 단일 yolo11s(0.547)가 앙상블(0.542)보다 근소 우위. **앙상블 강점=recall(더 많이 찾기)이지 위치 정밀도 아님.**
- 킬러 슬라이드: **DJI_0961_05_05** — 챔피언이 타워 세로 긴 균열을 통째 놓침 vs 앙상블이 잡음.
- 자료: `ensemble_wbf/ensemble_bars.png`, `ensemble_wbf/ensemble_qual_*.png`(6장), `ensemble_wbf/앙상블_결과_설명.md`.
- ★박정우 파트(D)와 겹침 = 의도된 것(각자 직접 해보는 학습 기회). 발표는 "최고 모델 경쟁"이 아니라 "팀이 넓게 탐색"으로 엮는다.

## F. 김민 — 증강 재학습 + 앙상블 백테스트 (2막) — 🔶 읽음 (지표 일부 미저장)

> 김민 깃허브(`Engineer3rd-Class-Kimmin/AIFFEL_QUEST_ENG` Main_Quest/MQ02) 노트북 3개를 읽어 정리(2026-07-02).
> ★**정정**: "외부데이터 학습"이라기보다 실제 내용은 **원본(kagglehub `wind-turbines-586x371` = 우리와 같은 Nordtank 출처) 재split + 증강 재학습**이 중심이고, 외부(roboflow **WTBD**)는 **테스트/백테스트용**으로 썼다.

**김민의 3단계 여정**

1. **first_yolo_model** — 우리 baseline을 본인 방식으로 변형. 원본 재split하되 **정상 이미지를 balanced(abnormal 2395 : normal 2400 ≈ 1:1)로 대량 포함**, dirt 층화. 여러 모델(yolo11n/v8s/v10s/12s) "balanced_fast"로 빠르게. → 우리 정상주입(train에만 소량 negative)과 대비되는 "정상 절반 섞기" 접근. **최종 정량지표는 results.png(이미지)로만** 남아 숫자 미확보.
2. **second_yolo_model** — recall 개선 시도. **원본 abnormal을 albumentations로 대규모 증강(목표 12,000장) + 정상 20%**, yolov8s.pt 전이학습. 시도 변천: 5:5 → 파인튜닝 → 최종 "증강 90% + 정상 10%". **recall 0.7 목표 미달**, GPU 없이 LMS/colab/kaggle 옮겨다녀 **최종 결과 파일 저장 못함**(학습곡선 이미지만).
3. **third_yolo_model** — **김민 yolov8s + 우리 5모델(11s/12s/v8s_norm1500/v10s_norm750 + rtdetr) = 6모델 앙상블(NMS, IoU 0.55)** 을 WTBD 라벨 100장에 백테스트. 정성적으론 "상용화 가능한 수준!"이라 판단했다가 → **정량 확인: recall 14%, mAP50-95 1% → "상용화 수준 아니다"로 정정("어???").**

**★우리 결과와의 연결 (발표 강력 재료)**
- 김민 3번(6모델 NMS, WTBD)의 **recall 14% / mAP50-95 1%** 는 **우리 WTBD 검증**(WBF 5모델, mAP50 0.06 · mAP50-95 0.027)과 **독립적으로 같은 결론** → **도메인갭이 특정 앙상블 방식(WBF/NMS)·특정 사람이 아니라 데이터 자체 문제**임을 두 경로가 확증.
- 김민의 "정성적으론 잘 잡는 것 같았는데 정량은 참담" = **"눈대중 착시 vs 지표의 진실"** 교훈(발표 회고 톤에 좋음, 개인 콕집기 아니라 팀 공통 배움으로).

**참고: 내가 이미 한 WTBD 일반화 검증(잣대 B)** — 우리 앙상블을 WTBD 161장에: mAP50 0.85→**0.06 붕괴**, 앙상블 recall 0.139→**0.429**(견고). 자료 `wtbd_generalization/WTBD_일반화_리포트.md`.

## H. rtdetr 환경 한계 분석 (2막, 김민욱) — ✅ 완료

- 로컬 GPU(Strix Halo APU, ROCm)에서 RT-DETR만 크래시 → `HSA_OVERRIDE_GFX_VERSION=11.0.0` + `MIOPEN_FIND_MODE=2`로 우회(에폭당 ~10분).
- ★발산 결론: lr 낮춰도(0.0008) 발산이 ep19→ep24로 "지연"만 되고 결국 터짐(train nan). **lr은 발산 시점만 조절, 근본 원인=환경(ROCm 우회 수치 불안정).** 높은 lr 일찍 터짐(빨리 배워 mAP 0.81) / 낮은 lr 늦게 터짐(느려 0.54) — 어느 쪽이든 발산.
- 교훈: ep19만 보고 "극복 성공" 오판할 뻔 = 한 에폭 단정 금물. **실패지만 "환경 한계를 실험으로 규명"** = 강한 회고 서사.
- 자료: `charts/4_training_curves.png`(발산 곡선), `MQ02_종합_히스토리.md` 환경 섹션.

## 종합 + 최종모델 + 회고 (3막) — 골격

- **상호보완 정리**: 앙상블(recall↑) ↔ 정상주입(FP↓) = 두 병목을 서로 다른 레버로 동시 공략. → 팀 최종모델 = (정상주입 pt들) + rtdetr 앙상블.
- **일반화 한 줄**: 외부 데이터(WTBD)에선 절대성능은 붕괴(도메인갭)하지만 앙상블 우위는 견고 → "단일 소스 한계 + 앙상블·다양데이터의 가치" 동시 증명.
- **회고 톤**: 각자 다른 레버를 직접 잡아 팀으로 넓게 탐색. 실패(rtdetr)도 규명하면 자산. (개인별 배움은 슬라이드에 콕 집지 않고 전체 톤으로.)

## 팀원에게 요청할 것 (복붙용 요약)

받을 내용을 팀원별로 한 번에 정리. 각자에게 이 항목들을 부탁하면 됨.

- **박정우** — baseline 앙상블: ①어떤 모델 조합 ②기준 pt(baseline/증강) ③앙상블 방법 ④mAP50·damage recall(잣대 표기) ⑤단일→앙상블 수치 ⑥그림. **(아직 자료 대기)**
- **이예령** — 타일화 전처리: ①타일화 의도·방법(왜 960², 원본 소스, overlap 여부) ②전/후 이미지 1~2장 ③본인이 학습했으면 지표. (blade30 우리 앙상블 평가는 완료)
- **김민** — 증강 재학습+앙상블: ①first/second **최종 정량지표**(results.png 속 수치나 best.pt) ②third 앙상블 recall 14%가 damage 기준인지 전체인지. (노트북 3개는 읽음)
- **최병철** — 영상→탐지 웹 서비스: ①탐지에 쓴 모델(우리 pt? Gemini? 자체?) ②AI Studio 제작·배포 과정 ③프레임 추출 처리 ④**데모 스크린샷/영상(백업 필수)** ⑤잘 되는/안 되는 케이스.

> ★공통 부탁: 지표는 **어느 잣대(공식 val / 자체 측정)인지**, **어느 split(test301인지 아닌지)** 꼭 같이. 그림은 원본 해상도로.
