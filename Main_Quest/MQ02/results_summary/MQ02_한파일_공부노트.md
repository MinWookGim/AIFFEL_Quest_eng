## 0. 문제가 뭐였나

드론이 찍은 풍력터빈 블레이드 사진에서 **오염(dirt)과 손상(damage)을 자동으로 찾는** 객체탐지(Object Detection) 문제다.
분류(사진 전체에 라벨 하나)와 다르게, 탐지는 "어디에(박스) 무엇이(클래스) 있다"까지 맞혀야 한다.

우리가 만든 첫 모델의 두 가지 실수가 이 프로젝트 전체의 출발점이다:
- 멀쩡한 블레이드 사진 200장 중 60장(30%)에서 "손상!" 헛경보 (잔디, 기둥을 손상으로 봄)
- 동시에 진짜 손상은 10개 중 3개를 놓침

현장으로 치면 "정비팀은 헛걸음하고, 진짜 손상은 방치"다. 이 둘을 줄이는 게 우리의 싸움이었다.

## 1. 데이터를 뜯어보면 (EDA)

- 전체 **13,470장** 중 라벨(정답 박스) 있는 건 **2,995장**뿐. 나머지 **10,475장(78%)은 "정상"**(아무 문제 없음)인데, 기본 코드는 이걸 통째로 버리고 있었다. 모델이 정상을 배운 적이 없으니 깨끗한 곳도 손상으로 본 것.
- 박스 수는 **dirt 581 : damage 8,770 = 약 1:15**. 오염이 아주 드물다.
- 그런데 **크기는 정반대**다 (직접 라벨을 세서 잰 것):
  - dirt 박스 중앙값 = **화면의 32%** (화면 3분의 1을 덮는 초대형 얼룩덩어리)
  - damage 박스 중앙값 = **화면의 0.15%** (약 18x18픽셀짜리 바늘)
- 한 줄 요약: **드물지만 큰 오염, 흔하지만 작은 손상.** 이 특징이 뒤에서 계속 중요해진다.
- 중앙값이 뭐냐면: 값들을 줄 세워 정확히 가운데 값. 평균과 달리 극단값 몇 개에 안 휘둘려서, "보통 크기"를 말할 때 정직하다. (예: 1,2,3,4,100의 평균은 22지만 중앙값은 3)

## 2. 성적표 읽는 법 — 지표 개념 (여기만 이해하면 숫자가 다 읽힌다)

### IoU = 박스가 맞았는지 재는 자
탐지는 박스가 "애매하게 맞을" 수 있어서, 정답 박스와 예측 박스의 **겹침 비율(IoU = 교집합/합집합)**로 판정한다.
IoU가 기준선(보통 0.5)을 넘으면 "맞은 탐지(TP)"로 인정.

### 정밀도(precision) / 재현율(recall) = 실수의 두 종류
- precision = 내가 "손상"이라 외친 것 중 진짜 비율 -> **헛것을 안 보는 능력**
- recall = 실제 손상 중 내가 찾아낸 비율 -> **안 놓치는 능력**
- 풍력터빈 점검은 "놓치면 큰일"이라 우리는 **recall을 우선**했다.

### mAP50 vs mAP50-95 = 채점 "자의 개수" 차이
- **mAP50** = IoU 자를 **0.5 하나만** 씀. 박스가 좀 헐렁해도 찾으면 인정 -> "손상을 **찾았나**"
- **mAP50-95** = 자를 0.50, 0.55, ... 0.95까지 **10개** 들이대 10번 채점해 평균 -> "박스를 **칼같이 그렸나**"
- 예: 정답과 IoU 0.72로 겹친 박스 하나 -> mAP50에선 만점, mAP50-95에선 자 10개 중 5개만 통과라 반쪽 점수.
- 주의: precision/recall(실수의 종류)과 mAP50/50-95(자의 개수)는 **서로 다른 축**이다. 어떤 자 위에서든 P와 R을 각각 잴 수 있다.

## 3. 공정 비교의 틀 — 왜 우리 숫자들은 서로 비교 가능한가

- 라벨 있는 2,995장을 **80/10/10**으로 잘라 train 2,395 / val 299 / **test 301**.
- 자를 때 **Dirt 층화(stratify)**: dirt가 1:15로 희귀해서 무작위로 자르면 한 통에 쏠려 평가가 운에 휘둘린다. dirt 포함 이미지를 세 통에 골고루(약 19%씩) 나눴다.
- **test301은 "금고에 넣은 수능 시험지"다.** 이후 모든 실험(5모델, 정상주입, 앙상블)이 **같은 301장**으로 채점됐고, 학습 재료(train)만 바뀌었다. 시험지가 같아야 점수 차이 = 모델 실력 차이가 성립한다.
- 학습 설정도 전원 동일: **50에폭 / imgsz 640 / batch 16 / seed 42.**
- imgsz란: 모델은 사진을 원본 크기 그대로 안 먹는다. 넣기 전에 정해진 한 변 크기(우리는 640)로 리사이즈한다. 우리 원본은 586x371.

## 4. 1차전 — 5모델 오디션 (각자 부딪히기)

후보를 하나만 찍지 않고 오디션을 열었다. YOLO 4세대(v8s/11s/v10s/12s) + 계열이 다른 트랜스포머 기반 rtdetr-l. 전부 s 체급, 같은 설정.

| 모델 (test301, 공식 채점) | mAP50 | mAP50-95 | dirt AP | damage AP | damage recall |
|---|---|---|---|---|---|
| **yolov8s (종합 승자)** | 0.863 | **0.627** | 0.969 | 0.757 | **0.707** |
| yolo11s | 0.855 | 0.613 | 0.954 | 0.756 | 0.670 |
| yolov10s | 0.846 | 0.597 | 0.955 | 0.736 | 0.666 |
| yolo12s | 0.866 | 0.617 | 0.972 | 0.760 | 0.665 |
| rtdetr-l (반쪽) | 0.800 | 0.440 | 0.852 | 0.748 | 0.674 |

발견 3가지:
1. **최신이 항상 낫진 않다** — 오래된 축인 yolov8s가 종합 승자.
2. **오염(dirt)은 다 잘 잡는다(AP 0.95+)** — 1:15로 드문데도. 이유는 1장에서 본 크기: 화면 3분의 1짜리 초대형 과녁이라 몇백 개만 봐도 배우기 쉽고 IoU 0.5 넘기기도 쉽다. **병목은 damage recall(0.71 안팎 = 30% 놓침)**.
3. **rtdetr은 우리 AMD GPU 환경에서 학습이 발산(nan)**해 반쪽 성적. lr을 낮춰도 발산 시점만 늦춰질 뿐 못 막았다 -> 원인은 환경(수치 불안정). "실패도 원인을 규명하면 자산"의 사례고, 이 낙제생은 뒤에 반전이 있다.

같은 시기 팀에서는 정상을 절반 섞은 변형 baseline, 공통 코드 재현(yolov10s mAP50 0.846 — 우리 결과와 일치) 실험이 병행됐다. **따로 돌렸는데 같은 병목**이 나온 게 중요했다.

## 5. 개선 1 — 정상 이미지 주입 (헛경보 줄이기)

**원리**: YOLO에서 라벨 파일이 "비어 있는" 이미지 = "이 사진엔 정답 박스가 0개"라는 뜻의 **정답**이다(배경 negative). 위조지폐를 감별하려면 진짜 지폐도 봐야 하듯, "문제 없는 경우도 있다"를 가르치는 것. 코드로는 정상 사진을 train에 복사하고 **빈 라벨 파일을 만들어주는 한 줄**이 전부다.
(★val/test는 1바이트도 안 바꿈 — 시험지 동일 = 공정 비교. 이 아이디어는 수업 때 신재욱 퍼실님이 가르쳐주신 내용에서 출발했다.)

**결과 (yolov8s 고정, 주입량만 변화 — 변수 격리)**:

| 주입량 | 헛경보율(정상 200장) | damage recall (test301, 공식) |
|---|---|---|
| 0장 (baseline) | 30% (헛박스 80개) | 0.707 |
| 250장 | 25% | 0.670 |
| 750장 | 16% | 0.705 |
| 1500장 | 17% | **0.722 (최고)** |
| 2500장 (train의 절반) | 14% | 0.689 (처음 꺾임) |

**읽는 법**: 헛경보는 꾸준히 줄고, recall은 1500장까지 유지·회복하다가 2500장에서 꺾인다. 정상을 너무 많이 주면 모델이 "겁쟁이"가 되어 진짜 손상도 조심스러워하는 것. **최적 지점 = 750~1500장** (헛경보 절반 + recall 유지).

## 6. 개선 2 — 앙상블 (놓침 줄이기)

### 용어부터
- **앙상블** = "여러 모델의 결과를 합친다"는 **전략**의 이름.
- **WBF(Weighted Boxes Fusion)** = 그 전략을 실행하는 **합치기 알고리즘** 중 하나. 여러 모델이 같은 물체에 그린 박스들(IoU 0.55 이상 겹치면 같은 물체로 판정)을 확신도 가중평균으로 하나로 합친다. **NMS 방식**(겹치면 최고점 하나만 남김)도 앙상블의 일종 — 팀에서 v8+v12 2모델 NMS 앙상블("돋보기+현미경": 저확신까지 넓게 줍는 v8 + 어텐션으로 정밀한 v12)이 먼저 신호탄을 올렸고, 그 문서가 촉매가 되어 5모델 WBF로 확장됐다.
- 재학습이 필요 없는 이유: 완성된 모델들의 **출력(박스 목록)만** 합치는 연산이라서. 함수 하나로 된다.

### 잣대가 두 개다 (우리 프로젝트 숫자 읽기의 최대 함정)
- **잣대A = 공식 채점기** `model.val()`: baseline·정상주입 숫자들(0.707, 0.722 등). 절대값으로 유효.
- **잣대B = 자체 채점기(conf 0.05)**: 앙상블은 "모델"이 아니라 "박스 목록"이라 model.val()에 넣을 주체가 없다. 그래서 박스 목록끼리 GT와 대조하는 채점기를 직접 만들었고, **단일 모델들도 같은 채점기로 다시 재서** 비교했다. 상대비교로만 유효.
- **★금지 문장: "recall이 0.72에서 0.94로 올랐다"** — 잣대가 다른 두 숫자를 이으면 안 된다. 맞는 문장: "같은 자체 채점기 기준, 단일 최고 0.856 -> 앙상블 0.944".

### 결과 (test301, 잣대B)
- 단일 최고(yolov8s_norm1500) damage recall **0.856** -> 4모델 YOLO 앙상블 **0.891** -> **5모델(+rtdetr) 0.944**
- 혼자서는 낙제생이던 rtdetr(발산 전 최고 가중치, 발언권 0.5로 참여)을 넣자 최고가 됐다. **원리가 다른 모델은 다른 실수를 해서, 팀으로는 서로 빈틈을 메운다.** (오디션에서 계열 다른 모델을 뽑아둔 이유)
- 정성 증거: DJI_0961 — 단일 최고 모델이 박스 0개로 통째로 놓친 세로 균열을 앙상블이 잡음.

### 정직하게 읽기 (스스로 검증한 것)
- conf 0.05는 관대한 문턱이다. WBF의 이득 자체가 저확신 박스들의 상호 확인에서 나오기 때문인데, 문턱을 올려 재채점해보면 격차가 줄고(0.25에서 동률) 0.5에선 단일이 낫다. 즉 **앙상블의 이득은 "후보를 다 띄우고 사람이 확인하는 스크리닝 운영점"의 이득**이고, 우리 용도가 정확히 그것이다.
- 문턱과 무관한 비교(AP = 곡선 전체의 넓이)로도 앙상블 승: damage AP 0.743 -> 0.811, mAP50 0.845 -> 0.863.
- mAP50-95는 단일(0.547)이 앙상블(0.542)보다 근소 우위 — WBF가 박스를 평균내며 위치가 살짝 뭉개지고, 앙상블이 새로 건진 어려운 손상일수록 박스가 헐렁해서다. **앙상블의 강점은 위치 정밀도가 아니라 recall**이라고 스스로 말하는 게 정확하다.

## 7. 시련 — 외부 데이터에선 무너진다 (도메인갭)

### 무슨 일이 있었나
잘 만든 것 같아서, 완전히 다른 곳에서 찍은 외부 데이터에 그대로 돌렸다:
- **WTBD** (Nature 논문 데이터, 1024², 6클래스 전부 손상 계열 -> damage로 통합, test 161장): mAP50 **0.85 -> 0.06 붕괴**
- **blade30** (Nature 논문, 근접 촬영 원본 5456²급, 960² 타일, dirt/damage 클래스 동일): mAP50 **0.02**, 특히 **dirt AP 0.000 (정답 23개 전부 놓침)**

### 도메인갭이란
**모델이 배운 세상(학습 데이터의 분포)과 새로 만난 세상이 달라서 성능이 무너지는 현상.** 능력을 상실한 게 아니라 환경이 바뀐 것 — 한국에서 운전 배운 사람이 영국 좌측통행에서 헤매는 격이다. 같은 도메인의 새 데이터(test301)에서는 0.85를 유지했으니 "못 배운 것"도 "과적합"도 아니다.

### 원인 규명 3종 세트 (우리가 제일 잘한 부분)
| 용의자 | 검증 실험 | 판정 |
|---|---|---|
| 해상도 | 추론 imgsz를 1024로 올려봄 | 오히려 더 나빠짐 -> 기각 (모델이 배운 "크기감"은 640 세상 기준) |
| 전처리(타일) | 타일 안 한 원본으로도 평가 | 똑같이 붕괴 -> 무죄 |
| 사람·방법 | 전혀 다른 방식의 6모델 NMS 앙상블로도 평가 | 동일 결론 -> 무죄 |

서로 다른 사람이 다른 방법으로 같은 답 -> **진범 = 데이터 그 자체.** 구체적으로 다른 것: 촬영 조건(원거리 586x371 vs 근접 5456²), 터빈 기종, 그리고 **라벨 기준**(무엇을 어디까지 손상/오염이라 박스 치는가 — 클래스 개수는 변환으로 맞춰도 이건 해소 불가).

### dirt가 0점이 된 원리 (데이터 부족이 아니다)
dirt는 집(test301)에선 AP 0.97로 1등이었다. 부족해서 못 배웠다면 집에서도 못 잡았어야 한다.
진짜 이유는 **박스 크기 관례가 통째로 뒤집혀서**다: 우리 dirt 중앙값 32% vs blade30 dirt 3.3~4.9% (7~10배 축소, 심지어 damage보다 작아져 크기 순위 역전). 모델이 배운 건 "오염"이라는 보편 개념이 아니라 "우리 데이터 라벨러가 크게 두른 얼룩 영역"이라는 한 데이터셋의 관례였고, blade30엔 그 템플릿에 맞는 물체가 아예 없었다. (+우리 데이터에선 dirt가 1:15로 드물다는 "감각"까지 배웠는데 blade30은 157:110으로 dirt 우세 — 감각도 어긋남)

### 그래도 안 무너진 것
붕괴 속에서도 **앙상블이 단일보다 손상을 2~3배 더 찾는다는 우위는 유지**됐다: WTBD에서 단일 recall 0.139 -> 앙상블 0.429 (3배). blade30에서도 같은 패턴(0.167 -> 0.405, 2.4배). 낯선 세상에서 앙상블이 더 견고하다.

## 8. 확장 갈래 — 영상과 서비스

- **증강 재학습**: 회전·밝기 변형으로 학습 사진을 약 12,000장 규모로 불리고 정상 10%를 섞어 재학습. recall 0.7의 벽은 못 넘었지만 "왜"까지 기록. 우리 5모델과 합쳐 6모델 백테스트로 이어짐.
- **영상 탐지 데모**: 프레임에서 터빈 영역(ROI)을 잘라 확대 -> 6모델 앙상블 탐지 -> 결과를 원본 좌표로 복원(zoom ROI auto-follow). 작은 손상 놓침 문제의 영상판 대응. 유튜브 영상은 또 다른 도메인이라 오탐·낮은 확신도 존재 — 되는 것과 안 되는 것을 그대로 보여주는 게 우리 방침.
- **웹서비스 + 검사기술 리서치**: 실제 현장은 아직 사람이 마우스로 박스 치고 눈으로 심각도를 판단하는 수동 검사(AS-IS). 우리의 한 걸음 = 영상 올리면 2초 간격 프레임 추출 -> 오염·손상 탐지하는 웹서비스(클라우드 배포). 더 멀리(TO-BE) = 표면 아래 손상은 RGB 카메라 한계라 열화상 등 다른 센서 필요 — 우리 도메인갭 결론과 정확히 이어진다.

## 9. 회고 — 세 가지 배움

1. **눈대중과 지표는 다르다.** 예측 그림으로는 잘 잡는 것 같아도 지표는 참담할 수 있다 (외부 데이터 백테스트에서 실제로 겪음).
2. **실패도 원인을 끝까지 파면 자산이 된다.** 학습이 계속 터진 rtdetr — 발산의 원인(환경)을 규명했고, 앙상블에서는 에이스가 됐다.
3. **낯선 현장까지 잘하려면 결국 다양한 데이터를 실제로 배워야 한다.** 도메인갭은 기법(해상도, 타일, 앙상블 방식)으로 우회가 안 됐다.

그리고 팀명 이야기: 실험은 정말로 팀원들 사이를 **빙글빙글** 돌며 완성됐다. 한 사람의 앙상블 문서가 다른 사람의 5모델 앙상블이 되고, 그 묶음이 또 다른 사람들의 외부 데이터 검증으로 돌아왔다.
