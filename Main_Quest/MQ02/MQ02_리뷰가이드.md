## 피어리뷰 PRT 문항별 확인 위치

리뷰어가 캡처 근거 찾기 쉽게, README의 PRT 5문항이 어디에 해당하는지 적어둔다. (전부 본체 노트북 안의 절 번호 기준)

| PRT 문항 | 어디를 보면 되나 |
|---|---|
| 1. 완성된 코드 제출 | 노트북 전체(4~12절, 실행 출력 포함). 최종 결과물 = 노트북 + `code/` 스크립트 + `presentation/slides_v3.pdf` |
| 2. 핵심 코드의 주석 | 노트북 9절 WBF 앙상블 셀(왜 자체 채점기가 필요한지 주석으로 설명), `code/ensemble_wbf_final.py`(파일 머리에 잣대 주의 docstring), 8절 정상 주입의 빈 라벨 코드 |
| 3. 디버깅·새로운 시도 | 7절 rtdetr 학습 발산 원인 규명, 8절 주입량 스윕(변수 격리), 9절 "정직하게 읽기"(conf 문턱 스윕 자기검증), 10절 도메인갭 원인 규명 3종. 팀원별 추가 실험은 `team/` 02~05 |
| 4. 회고 | 노트북 12절(세 가지 배움) + 부록 A(숫자 총정리)·부록 B(문서 지도). 전체 실행 흐름은 노트북 맨 위 개요와 2절 실행 설정 |
| 5. 간결·효율 | 노트북 5절 split 함수(팀 공통 코드로 재사용), `code/ensemble_wbf_final.py`가 `ensemble_wbf_rtdetr.py`의 채점 함수를 import로 재사용(중복 방지 + 동일 잣대), `code/` 스크립트들의 함수화 |
